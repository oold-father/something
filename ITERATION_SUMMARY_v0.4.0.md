## 一、迭代目标

优化文件监控功能，确保文件管理与监控范围保持一致。

### 1.1 主要目标
管理监控文件夹时，确保文件范围仅在监控列表内，移除操作同步更新文件列表。

## 三、任务拆分

### Task #1：移除监控文件夹时同步移除文件

**描述**：当用户从监控列表中移除某个文件夹时，需要同时移除该文件夹下的所有文件，确保文件范围与监控范围一致。

**技术要点**：
- 前端：WatchedDirectories 组件移除监控目录后端
- 后端：修改 remove_watched_directory 接口，先删除文件再删除目录
- 数据库：使用文件路径前缀匹配删除相关文件

**验收标准**：
- [ ] 移除监控目录后，文件列表中不再显示该目录下的文件
- [ ] 标签关系正确清除
- [ ] 无孤儿数据残留

---

### Task #2：移除监控文件夹时清除标签关系

**描述**：移除监控文件夹时，需要同步清除文件与标签的关联关系，避免残留的标签引用。

**技术要点**：
- 后端：利用外键约束（ON DELETE CASCADE）自动清理
- 确保 file_tags 表的外键配置正确

**验收标准**：
- [ ] 移除监控目录后，相关标签关系被清除
- [ ] 无标签引用错误
- [ ] 数据库完整性保持

---

### Task #3：监控文件夹扫描时检测文件删除

**描述**：当监控文件夹进行周期性扫描时，如果发现文件已被物理删除，需要从文件列表中同步移除，并清理相关标签关系。

**技术要点**：
- 后端：scanner.rs 中检查文件是否存在
- 不存在的文件标记为 deleted 状态或直接删除
- 触发前端文件列表刷新

**验收标准**：
- [ ] 文件删除后自动从列表中移除
- [ ] 标签关系自动清理
- [ ] 无孤儿文件记录残留
